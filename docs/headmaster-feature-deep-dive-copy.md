# Headmaster Feature Deep-Dive Copy

## The System Behind Headmaster
Headmaster is more than a chat box. It is a connected workspace where memory, tools, skills, automations, channels, specialist agents, and approvals work together.

Use this section to help visitors understand the system behind the product without jargon.

---

## 1) Persistent Memory

**Feature name:** Persistent Memory  
**Plain-English name:** Long-term memory for how your organization works  
**One-line explanation:** Headmaster remembers the preferences, decisions, workflows, tone, and context your team uses again and again, so people do not have to repeat themselves.

**Website section copy:**  
Headmaster keeps track of how your organization works over time. It remembers preferences, recurring instructions, previous outputs, tone, decisions, document context, and the small details that make everyday work smoother. Instead of starting from zero every time, your team gets continuity.

**Everyday analogy:**  
A long-term notebook for how your organization works.

**Example in a school:**  
It remembers the school’s report format, parent communication tone, grading language, and previous progress notes.

**Example in a business:**  
It remembers client preferences, reporting format, proposal style, recurring tasks, and previous decisions.

**Example microcopy for card:**  
Keeps your organization’s context in one place.

**What NOT to say:**  
- “It watches everything.”
- “It stores surveillance data.”
- “It remembers you personally.”
- “Creepy long-term tracking.”

---

## 2) Self-Learning / Skills

**Feature name:** Self-Learning / Skills  
**Plain-English name:** Reusable playbooks  
**One-line explanation:** When Headmaster solves a repeated workflow, it can turn that process into a reusable way of working.

**Website section copy:**  
Headmaster does not need every workflow explained from scratch forever. When a process repeats, it can remember the structure, improve the steps, and turn the work into a reusable playbook. That means less re-explaining, more consistency, and faster results over time.

**Everyday analogy:**  
A staff member who writes down the best way to do a task after doing it once.

**Example in a school:**  
After generating progress reports several times, it learns the structure and turns it into a repeatable reporting workflow.

**Example in a business:**  
After preparing weekly client updates, it learns the format and can prepare the next one faster with less explanation.

**Example microcopy for card:**  
Turns repeated work into a repeatable playbook.

**What NOT to say:**  
- “We train the model.”
- “Fine-tuning.”
- “Machine learning pipeline.”
- “It magically gets smarter.”

---

## 3) Tools

**Feature name:** Tools  
**Plain-English name:** Approved work systems  
**One-line explanation:** Tools are the approved systems Headmaster can use to do real work, like reading documents, checking calendars, drafting messages, searching files, or interacting with internal systems.

**Website section copy:**  
Headmaster is not limited to conversation. It can work with the approved systems your organization already uses, such as documents, inboxes, calendars, databases, browsers, file libraries, and internal workflows. That means it can help prepare real work, not just talk about it. Access stays permission-based and controlled.

**Everyday analogy:**  
Hands and instruments, not just a brain.

**Example in a school:**  
It can read lesson documents, check attendance sheets, prepare report drafts, and use approved templates.

**Example in a business:**  
It can search documents, prepare reports, summarize client notes, draft emails, and check project files.

**Example microcopy for card:**  
Works inside the systems your team already trusts.

**What NOT to say:**  
- “It has unlimited access.”
- “It can do anything anywhere.”
- “It hacks into your systems.”
- “MCP-powered tool orchestration.”

---

## 4) Gateways / Channels

**Feature name:** Gateways / Channels  
**Plain-English name:** Different ways to reach the same workspace  
**One-line explanation:** Headmaster can be reached from the places work already happens, like a dashboard, email, Slack, WhatsApp, Telegram, Discord, or other approved channels.

**Website section copy:**  
Your team should not have to live inside one dashboard to get work done. Headmaster can be reached from different channels, while keeping the same memory and workflow history behind the scenes. That means people can start work, check status, or receive updates in the place that is most natural for them.

**Everyday analogy:**  
Different doors into the same office.

**Example in a school:**  
A teacher can receive a draft update by email, while admin checks workflow status in the dashboard.

**Example in a business:**  
A manager can start a task from chat and later review the output from the web dashboard.

**Example microcopy for card:**  
Work from the channel your team already uses.

**What NOT to say:**  
- “Gateway process.”
- “Message bus.”
- “Channel routing architecture.”
- “Only available through our proprietary interface.”

---

## 5) Automations

**Feature name:** Automations  
**Plain-English name:** Scheduled work  
**One-line explanation:** Automations are recurring tasks that wake up on schedule and run without someone starting them manually each time.

**Website section copy:**  
Headmaster can take on recurring work that should happen regularly: weekly summaries, daily checks, scheduled reports, reminders, and other repeating tasks. It starts itself at the right time, follows the workflow, and prepares the output for review. Sensitive actions still wait for approval.

**Everyday analogy:**  
A task that wakes up at the right time.

**Example in a school:**  
Every Friday, Headmaster drafts parent updates for review.

**Example in a business:**  
Every Monday morning, Headmaster summarizes project activity and prepares client updates.

**Example microcopy for card:**  
Runs the work that should happen on schedule.

**What NOT to say:**  
- “Cron-based automation.”
- “Always unattended.”
- “It sends everything automatically.”
- “Set it and forget it forever.”

---

## 6) Specialist Agents

**Feature name:** Specialist Agents  
**Plain-English name:** Focused helpers  
**One-line explanation:** Headmaster can bring together focused helpers for different parts of a task, such as research, writing, reviewing, checking, formatting, or organizing.

**Website section copy:**  
Some work is better handled as a team. Headmaster can break a task into smaller parts and use focused helpers for each step. One can research, another can write, another can check details, and another can format the final output. This keeps work moving while each part gets the right kind of attention.

**Everyday analogy:**  
A small team assembled for one workflow.

**Example in a school:**  
One helper drafts student comments, another checks sensitive language, and another formats the report.

**Example in a business:**  
One helper researches, another writes the proposal, another checks the details, and another formats the final document.

**Example microcopy for card:**  
Brings the right helper to the right part of the job.

**What NOT to say:**  
- “Subagents.”
- “Distributed autonomous swarm.”
- “Agentic orchestration layer.”
- “Mini-bots.”

---

## 7) Approvals

**Feature name:** Approvals  
**Plain-English name:** Human sign-off  
**One-line explanation:** Headmaster can prepare work, but important actions pause until a person approves, rejects, or requests changes.

**Website section copy:**  
Headmaster is designed to keep humans in control. It can prepare emails, reports, edits, and other actions, but anything sensitive can pause for review before it goes out. That gives teams speed without giving up oversight.

**Everyday analogy:**  
A final signature before anything leaves the building.

**Example in a school:**  
Parent emails are drafted but never sent until a teacher or admin approves them.

**Example in a business:**  
Client reports can be prepared automatically, but delivery waits for manager approval.

**Example microcopy for card:**  
Nothing sensitive moves without review.

**What NOT to say:**  
- “Fully autonomous sending.”
- “No human needed.”
- “Bypasses review.”
- “It acts on its own.”

---

## 8) Documents / Knowledge Base

**Feature name:** Documents / Knowledge Base  
**Plain-English name:** Your organization’s library  
**One-line explanation:** Headmaster can work from the documents your team already trusts, including policies, templates, notes, reports, SOPs, and brand materials.

**Website section copy:**  
Every organization has important material it relies on. Headmaster can use that material as the source of truth: policies, templates, reports, meeting notes, brand guidelines, lesson plans, contracts, and more. Instead of guessing, it works from what your organization has already approved and stored.

**Everyday analogy:**  
A library Headmaster can read from before doing work.

**Example in a school:**  
It uses school policies, report templates, lesson plans, and student notes.

**Example in a business:**  
It uses proposals, contracts, meeting notes, SOPs, and brand guidelines.

**Example microcopy for card:**  
Uses the documents your team already relies on.

**What NOT to say:**  
- “It knows everything.”
- “It invents policies.”
- “It replaces your document system.”
- “It hallucinates answers from nowhere.”

## 9) Runs / Workflow Timeline

**Feature name:** Runs / Workflow Timeline  
**Plain-English name:** A visible trail of work  
**One-line explanation:** Each run shows what happened, what was used, what was created, and what still needs review.

**Website section copy:**  
Headmaster should not be a black box. Each workflow leaves behind a clear trail of what was done, what information was used, what draft or output was created, and what is still waiting for review. That makes it easy to trust the process and pick up where work left off.

**Everyday analogy:**  
A receipt for the work.

**Example in a school:**  
A report run shows which notes were used, which drafts were created, and which items need review.

**Example in a business:**  
A client update run shows documents checked, the draft created, and the approval status.

**Example microcopy for card:**  
See what happened, what changed, and what’s waiting.

**What NOT to say:**  
- “Logs” as the main headline.
- “Debug output.”
- “System trace.”
- “Invisible backend records.”

---

## 10) Private Workspace / Deployment

**Feature name:** Private Workspace / Deployment  
**Plain-English name:** Each organization gets its own space  
**One-line explanation:** Every organization can have its own private workspace with its own context, workflows, users, tools, and brand settings.

**Website section copy:**  
Headmaster is built so each organization can have its own space. That means private context, separate workflows, separate memory, separate users, separate tools, and branding that matches the organization if needed. Schools, clinics, agencies, and businesses can all work in their own environment without mixing with others.

**Everyday analogy:**  
Each organization gets its own office, not a shared desk.

**Example in a school:**  
A school’s teachers, templates, documents, workflows, and parent communication rules stay in its workspace.

**Example in a business:**  
A company’s clients, reports, documents, and internal workflows stay separate.

**Example microcopy for card:**  
A private workspace for each organization.

**What NOT to say:**  
- “Shared public instance.”
- “Everyone’s data in one bucket.”
- “Generic demo environment.”
- “Same workspace for all customers.”
